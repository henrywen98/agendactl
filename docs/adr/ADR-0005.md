# ADR-0005: 首批先做 Calendar（非 reminders）

- 状态：accepted
- 日期：2026-06-25
- 上下文：CLI-only 薄设计（[ADR-0001]..[ADR-0004]）尚未被任何真实代码验证。需先拿一个 App 跑通全链路再克隆。我（griller）推荐先做最干净的 reminders；Henry 否决，选 Calendar。
- 决策：**首批先做 Calendar**。理由（事后看，Henry 的反直觉选择更狠）：Calendar 是三者**最难**——首次 `-10000` 需 `open -a Calendar` 预热 TCC（§9.7）、`whose` 对日期不稳（§9.4）。用最坏情况去**压测**「把复杂度全塞进 CLI、不漏给 agent」（[ADR-0002]）这一论点；扛住 Calendar，reminders/notes 即降维。
- 后果：P1 范围 = `bin/apple calendar` 全套（calendars / list-events / create-event / update-event / delete-event）。reminders / notes 待 Calendar 验证后**原样克隆**模式。定位策略见 [ADR-0006]。
