# ADR-0003: 查询 = 固定粗筛 flag + agent 上下文内过滤

- 状态：accepted
- 日期：2026-06-25
- 上下文：[ADR-0001] 砍了 `--filter` DSL，[ADR-0002] 又删了逃生舱——招牌复合查询（「今天到期 + 优先级≥5 + 未完成」）**无处可去**。数据量是真实约束：Calendar 1626 事件、Notes 1004 笔记，全量裸返回又慢又烧 token。
- 决策：`list` 家族提供**固定粗筛 flag**（`--due` / `--status` / `--from` / `--to` / `--search` / `--limit`），返回 JSON；复合/长尾过滤由 **agent 在上下文内**完成。大列表**必须**能用 flag 收窄，避免 token 爆炸。
- 后果：不实现 `--filter` DSL，也不做无脑全量裸返回。复合查询从「一次接口搞定」改为「flag 粗筛 + agent 精筛」。flag 覆盖约 80% 常见过滤。
- 回退：若复合查询日后高频且痛，引入**极简** `--filter`（只 `$eq/$gte/$contains` 几个算子）——可逆，复杂度回升有限。
