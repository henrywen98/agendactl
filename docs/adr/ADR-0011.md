# ADR-0011: 对外发布不做 MCP server（重申 0008，升级理由）

- 状态：accepted（重申 [ADR-0008]；本会话一度决定「加可选 MCP 适配层」，经竞品分析后撤回）
- 日期：2026-06-28
- 上下文：对外开源时一度考虑加一层**可选** MCP 适配器,触达不能跑 shell 的 GUI agent(Claude Desktop / Codex desktop)。原型落地并实查竞品后推翻:
  - **Apple 日历/提醒的 MCP server 已饱和**:`supermemoryai/apple-mcp` ★3000+,另有十余个(FradSer/orchard/che-ical/Krishna/griches/macuse…),且功能更全(recurring / location / subtask / 批量)。以功能更少的迟到者进这个赛道 = 最弱位置。
  - **能跑 shell 的 agent 不需要 MCP**:pi / Claude Code / Cursor / Gemini CLI / cron / 脚本都直接调 CLI,MCP 对它们零增益。
  - **skill 本身就是给 agent 的分发渠道**(VoltAgent/awesome-agent-skills、ComposioHQ/awesome-claude-skills 等),而日历/提醒类 **skill 几乎空白**——蓝海。MCP 只是给 GUI app 的平行冗余渠道。
- 决策:**不自建 MCP server / 适配器**;定位为 **agent skill + CLI**。删除会话中曾加的 `mcp/` 目录。
- 后果:
  - 重申 [ADR-0008] 的「不做 MCP」结论(理由从「pi 内部约束」升级为「竞品饱和 + skill 才是分发渠道」)。
  - 范围:只服务能跑 shell 的 agent;跑不了 shell 的纯 GUI 客户端(如 Claude Desktop)**有意不覆盖**——那是 MCP server 的地盘。
  - 推广主投 **awesome-skills 列表**,而非 MCP 目录。
