# ADR-0011: 对外发布新增可选 MCP 适配层（反转 0008 的对外部分）

- 状态：accepted（修订 [ADR-0008] 的对外部分）
- 日期：2026-06-28
- 上下文：v2.0 对外开源。[ADR-0008] 决定「不自建 MCP server」，但其理由是 **pi 不支持 MCP + EventKit 直连开销更低**——这是 **pi 内部约束**，不是「MCP 不好」。对外发布后受众变了：不能跑 shell 的 GUI agent（Claude Desktop / Codex desktop）只能通过 MCP tool 拿能力，而 MCP 目录 + 桌面客户端正是这个生态最大的曝光与采纳入口。维持「绝不做 MCP」等于主动退出该渠道。
- 决策：新增**可选** `mcp/`——一个**薄 stdio MCP server**（Node）包 `skills/ekctl/scripts/ekctl`，把各命令暴露成 tools。CLI 仍是**单一事实源（SSOT）**；适配器**零业务逻辑**：只「拼 ekctl 参数 → spawn → 透传 stdout 裸 JSON → 非零退出码转 MCP error」。能跑 shell 的 agent（Claude Code / Cursor / pi / 脚本 / cron）继续直调 CLI，不经 MCP。
- 后果：
  - 反转 [ADR-0008] 的「对外不做 MCP」部分；pi 内部仍可只用 CLI，[ADR-0008] 的底座结论（纯 EventKit）不受影响。
  - 打开 MCP 目录（awesome-mcp-servers / smithery / mcpmarket / glama）与 Claude Desktop / Codex 受众。
  - 维护面 +1（Node 薄层），但因零业务逻辑、随 CLI 演进自动跟上；授权、时区、稳定 id、`end>start` 等仍只在 CLI 内一处。
  - `mcp/` 默认用 `skills/ekctl/scripts/ekctl`，可用 `EKCTL_BIN` 覆盖。
