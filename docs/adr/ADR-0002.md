# ADR-0002: agent 契约 = CLI-only，删除面向 agent 的逃生舱

- 状态：accepted
- 日期：2026-06-25
- 上下文：v1.0 的复杂度（3 次降级纪律 §2.2/§8.1、`menu`、§9/§10 JXA 陷阱、要 agent 读懂 JXA schema §8）几乎**全为「逃生舱」服务**——即「CLI 不够时让 agent 现场写 JXA」。Henry 提案「我建 CLI，agent 只调 CLI」。审讯指出：删掉逃生舱，agent 的世界瞬间清爽，复杂度只是移进 CLI（Henry 测一次）。
- 决策：agent **永远只调 CLI，不暴露任意 JXA 通道**。CLI 没覆盖的需求 → agent 告诉用户「做不了」或记录为「待 Henry 加命令」。`apple run` 仍可存在，但作为 **break-glass**（Henry / CLI 测试用），不写进 `SKILL.md`、不进 agent 工作流。
- 后果：删除 3 次降级协议、`apple menu`、§9/§10 面向 agent 的陷阱说明、JXA schema 教学（§8 对 agent 不可见，仅 CLI 实现者读）。`SKILL.md` 大幅瘦身。**能力封顶 = CLI 覆盖面**；缺失需求靠 Henry 加子命令补。
- 风险：开放式「生活自动化」长尾需求遇 CLI 未覆盖时只能「做不了」，依赖 Henry 在环。接受为 v1 取舍——**可逆**：日后可把 break-glass 升级为文档化逃生舱。
- 相关：[ADR-0001]（薄壳）、[ADR-0003]（查询如何在无逃生舱下表达）。
