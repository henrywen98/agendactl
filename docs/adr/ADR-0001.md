# ADR-0001: bin/apple 从「厚抽象层」降为「薄壳 + 透传」

- 状态：accepted
- 日期：2026-06-25
- 上下文：Henry 质疑 v1.0「太复杂」，并问「系统已给接口，为何不用系统的」。审讯澄清：v1.0 的底座**本就是** `osascript -l JavaScript` 调 Apple 的 AppleScript 字典（sdef）——那就是系统接口。`bin/apple` 不是另起炉灶，是其上的糖。真正有争议的不是「系统 vs 自建」，而是**这层糖该有多厚**。
- 决策：保留 osascript/JXA 底座。`bin/apple` 降为**薄壳**：只做几个已验证的参数化 CRUD 稳函数 + 透传，不做查询引擎。砍掉 v1.0 的：`--filter` MongoDB 风格 DSL（§2.6/§4.4）、`apple menu`（§5.4）、3 次降级协议（§2.2/§8.1）、probe 5min 缓存（§5.3）。
- 后果：删去对应章节与 `lib/std.js` 的 filter 引擎。面向 agent 的复杂度从「厚菜单 + 纪律」降为「薄命令 + `--help`」。能力上限议题转入 [ADR-0002]。
