# ADR-0008: 范围收缩至 Calendar + Reminders（去掉 Notes）→ 底座统一为纯 EventKit

- 状态：accepted（simplifies [ADR-0007]：底座分裂 → 单一 EventKit）
- 日期：2026-06-25
- 上下文：Henry 提「先不管 Notes，只管日历和提醒事项」。
- 关键观察：**EventKit 恰好覆盖 Calendar（`EKEvent`）+ Reminders（`EKReminder`），正好等于收缩后的全部范围**。Notes 是三个 App 里唯一只能走 JXA 的（无公开框架）。去掉它后，[ADR-0007] 的「底座分裂」坍缩为**单一纯 EventKit 底座，JXA 彻底退出**。
- 决策：
  - 首批范围 = **Calendar + Reminders**。Notes **搁置**（非删除——日后若做需单独引入 JXA 底座，见 [ADR-0007]）。
  - 底座 = **纯 EventKit**（无 JXA、无脚本字典、无 §9 坑）。
  - §9 / §10 的 JXA 陷阱章节对本范围**完全不适用**，实现者无需读。
- 后果：
  - 架构进一步简化：单框架、单授权模型、单查询机制（`NSPredicate`）、单身份方案（stable identifiers）。
  - `.pi/skills/notes/` 搁置；glossary 的 Notes 软删 / Account 等条目移出当前范围。
  - 重开 [ADR-0007] 留下的「CLI 语言 / 形态」决策，且 calculus 变了：**不再为 Notes 保留 JXA 运行时**——纯 EventKit 下「Swift 原生授权」vs「osascript ObjC 桥」的取舍更对等（osascript「单文件覆盖 Notes」的优势消失）。
