# ADR-0009: CLI 形态 = Swift 单文件脚本（纯 EventKit）

- 状态：accepted
- 日期：2026-06-25
- 上下文：[ADR-0008] 定底座为纯 EventKit 后重开语言选择。对比「Swift 单文件脚本」vs「osascript-JXA + EventKit 桥」（含同一操作的真实代码对照）。toolchain 实测：swift 6.2.3 在 `/usr/bin/swift` 且能当脚本直跑；pyobjc-EventKit 未装（Python 路线要 pip 依赖）。
- 决策：**Swift 单文件脚本**（`bin/apple`，`#!/usr/bin/swift`）。先脚本跑；介意启动延迟（脚本模式 ~1–2s）再一行 `swiftc bin/apple -o bin/apple` 编成二进制（<50ms）。
- 理由：
  - Notes 去掉后（[ADR-0008]），osascript「单运行时覆盖 Notes」的优势消失。
  - Swift 原生 EventKit 授权（`requestFullAccessTo...` + semaphore 同步等）**干净**，绕过 osascript 实测的 events full-access 握手 `done:false` 坑。
  - 类型安全 + 编译期查错 → 可维护性远高于 ObjC 桥的 `$.`/`ObjC.unwrap`/手动 runloop。
- 后果：`bin/apple` = 一个 `.swift` 文件。`lib/std.js`（JXA helper）不再需要。无新依赖（swift 已在）。
