# ADR-0012: 形态 = 单 skill 内置预编译二进制；改名 macctl → agendactl（反转 0010）

- 状态：accepted（反转 [ADR-0010] 的两-skill 拆分与命名）
- 日期：2026-06-28
- 上下文：对外开源 + 「装了 skill 即用、任意机器都能跑」目标,逼出两处重定:
  1. **命名**:`macctl` 在 GitHub 撞 6 个仓库;改 `ekctl` 后又发现撞 `schappim/ekctl`(同名同概念的 EventKit 日历/提醒 CLI)。两次都因**没先查名字占用**。
  2. **形态**:[ADR-0010] 把 calendar/reminders 拆两个 skill 共享 `bin/macctl`。一旦把二进制**内置进 skill**(装了 skill 即用、不依赖单独安装),共享就别扭——独立装其一时谁带二进制?
  3. **运行时**:`#!/usr/bin/swift` 脚本依赖 Xcode 工具链在场、每次 ~0.3s、二进制每重编一次 TCC 身份就漂。
- 决策:
  - 改名 **`macctl` → `agendactl`**(agenda = 日程 + 待办;`*ctl` 约定;**改名前已查 GitHub 零撞名**、可独占搜索词)。
  - 形态 = **单个 `agendactl` skill**(`skills/agendactl/`),同时覆盖 calendar + reminders,**内置预编译 universal2(arm64 + x86_64)ad-hoc 签名二进制** `scripts/agendactl`;旁附源码 `scripts/agendactl.swift` + `scripts/build.sh`(SSOT、可审计、可重建)。
  - 防误触发(calendar vs reminders)改由 `SKILL.md` 的 `description` 路由,不再靠拆 skill。
- 后果:
  - 反转 [ADR-0010] 的两-skill 拆分:装一个 skill 即全功能,二进制不重复。
  - **预编译胜过 swift 脚本**:无工具链依赖、~0ms 启动、ad-hoc 签名 → Apple Silicon 可执行 + TCC 身份稳定(`lipo` 合并破坏签名,故 `build.sh` 内 lipo 后必 `codesign -s -` 重签)。`git clone` 不打 quarantine,clone 即跑。
  - 首次 TCC 授权仍不可消除(EventKit 隐私边界,需 GUI 会话点一次允许),退出码 3 + hint 引导。
  - 目录 `bin/`、`lib/`、`mcp/`(见 [ADR-0011])删除;`.pi/skills/{calendar,reminders}/` 合并为 `skills/agendactl/`。
  - 当前态文档(ARCHITECTURE / glossary / ADR 索引)同步为 `agendactl`;[ADR-0001..0011] 里的 `macctl` / `ekctl` / `bin/macctl` **读作** `agendactl` / `skills/agendactl/scripts/agendactl`(历史不逐一改写)。
  - **教训**:命名前必先查 GitHub 名占用——`macctl`、`ekctl` 两次踩坑。
